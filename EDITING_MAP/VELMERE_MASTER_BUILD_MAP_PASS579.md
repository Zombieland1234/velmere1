# Velmère master build map — continuation after PASS579

This map preserves the full product split. Lens/PDF, Shield, Shield Map, VLM Brain and Real Markets remain independent public surfaces connected by shared evidence contracts; they must not collapse into one generic AI panel.

## A. Velmère Lens / PDF

- **PASS580 — PDF visual-regression fixtures:** deterministic PL/DE/EN fixtures for short, normal and overloaded reports.
- **PASS581 — page compositor:** move oversized sections to the next page before render; never clip or overlap.
- **PASS582 — source citation rail:** compact source IDs in Reader with full source details opened on demand.
- **PASS583 — download parity gate:** pixel/layout parity contract between browser Reader and generated PDF.
- **PASS584 — PDF accessibility:** heading order, readable contrast, selectable text and keyboard page navigation.

## B. Velmère Shield charts and analysis

- **PASS585 — shared crosshair inspector:** identical OHLC/time/volume inspection in Shield and Real Markets.
- **PASS586 — mobile chart gestures:** one-finger pan, anchored pinch zoom, no page-scroll theft outside chart focus.
- **PASS587 — chart viewport persistence:** remember timeframe/zoom per instrument without stale-provider contamination.
- **PASS588 — tier evidence manifests:** enforce 10 / 14 / 20 distinct verified fields for Basic / Pro / Advanced.
- **PASS589 — source freshness scheduler:** revalidate stale data without flashing or replacing confirmed values with placeholders.

## C. Shield Map / Orbit 360

- **PASS590 — snapshot replay timeline:** compare multiple prior snapshots, not only previous vs current.
- **PASS591 — node virtualization:** render only visible/nearby Orbit nodes under mobile GPU pressure.
- **PASS592 — evidence-path highlighting:** selecting a node highlights only its source and dependency path.
- **PASS593 — Map keyboard model:** roving focus, arrow navigation, Enter detail and Escape close.
- **PASS594 — calm-risk transitions:** animate state change only; no decorative motion when evidence did not change.

## D. VLM Brain

- **PASS595 — reasoning graph compression:** group repetitive evidence into source families before 3D render.
- **PASS596 — confidence limiter lens:** visually show which missing proof caps the final confidence.
- **PASS597 — contradiction resolution replay:** before/after evidence path with source timestamps.
- **PASS598 — low-power renderer:** static SVG/canvas fallback with identical information hierarchy.
- **PASS599 — brain-to-PDF handoff:** export the same verified decision brief without regenerated random wording.

## E. Real Markets

- **PASS600 — adapter truth matrix:** live / stale / delayed / fallback status per instrument and provider.
- **PASS601 — exchange-health adapters:** withdrawals, API, depth and reserve fields remain `unknown` until sourced.
- **PASS602 — catalogue density modes:** comfortable and compact layouts with the same information contract.
- **PASS603 — asset identity registry:** verified logos, names, exchange/market and canonical instrument IDs.
- **PASS604 — cross-asset comparison:** normalized percentage/volatility comparison without mixing incompatible units.

## F. Shared AI and evidence engine

- **PASS605 — claim provenance IDs:** every public AI sentence links to facts, inference or missing-data reason.
- **PASS606 — deterministic narrative composer:** identical evidence produces stable wording across Reader, Shield and Map.
- **PASS607 — uncertainty grammar:** locale-specific wording for confirmed, inferred, stale, conflicted and unavailable facts.
- **PASS608 — hallucination stop-gate:** block publication when a claim has no source or declared inference path.
- **PASS609 — evidence retention policy:** define what is cached, refreshed, expired and excluded from public reports.

## G. UI, motion and performance

- **PASS610 — interaction latency budget:** record input-to-render timing for search, chart, drawer and modal open.
- **PASS611 — unified motion tokens:** duration/easing/distance by interaction importance and reduced-motion mode.
- **PASS612 — mobile safe-area sweep:** header, sheets, PDF Reader, chart controls and bottom rails.
- **PASS613 — surface containment audit:** stop hidden modals, 3D canvases and backdrops from repainting the page.
- **PASS614 — visual hierarchy QA:** automated checks for clipped labels, overlapping panels and unreadable contrast.

## H. Security and release quality

- **PASS616 — API payload schema gates:** reject depth, locale, source and identity mismatches at route boundaries.

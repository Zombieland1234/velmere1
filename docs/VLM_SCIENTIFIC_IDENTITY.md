# VLM Scientific Identity

This document prevents the AMU / rho / prime-field layer from being mistaken for wallet cryptography.

## AMU

`AMU = 3162.27766 = sqrt(10^7)`

AMU is not cryptography. It is the visual carrier constant of the VLM Protocol Lab: the central pulse used by the interface to organize rings, node motion and access-gate visuals.

## Rho

`rho = 1.324717957244746`

Rho is the plastic constant. In the UI it spaces harmonic rings and gives the Pro interface a consistent growth rhythm.

## Prime Field

Prime nodes are visual markers only. They do not create wallets, secure funds, replace Ethereum/Base cryptography or prove contract safety.

## Access Formula

`AccessScore = f(address, balance, network, rules)`

This is a UI eligibility model. It can explain how a public address, token status, network and registry rules produce an access result. It is not a cryptographic proof and it does not predict price.

## Wallet Genesis

Wallet Genesis is educational animation only. It never creates a wallet, never generates keys, never shows seed phrases and never stores private wallet material. Real wallets are created by trusted wallet apps such as MetaMask or Phantom.

## Real Security

Real VLM security must come from wallet custody, standard contract patterns, fixed supply, no owner mint, no blacklist, no honeypot, multisig administration, testnet validation, static analysis and an independent audit before any production activation.

Public name: **VLM Protocol Lab**. Internal research name may appear only as a small technical note: **Bajak Protocol**.

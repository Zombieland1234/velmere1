# Velmère PASS 121 — Commerce Trust Launch Protocol

## Purpose
Velmère must remain clothing-commerce first. VLM, Shield and Square can build depth, but the product page must earn trust through fit, material, delivery, returns and checkout clarity.

## Launch audit model
The commerce launch audit separates:
- preview products,
- active sale-ready products,
- disabled checkout,
- missing fulfilment mapping,
- missing images,
- missing variants,
- unavailable variants,
- incomplete PL/EN/DE copy,
- manual provider confirmation.

## Current product state
The first drop products are intentionally `coming_soon` with disabled fulfilment. That is acceptable only if the UI labels them as preview and does not fake checkout readiness.

## Before public sale
Each product needs:
- final material/composition,
- fit note,
- size chart,
- care instructions,
- provider/fulfilment confirmation,
- stock/variant availability,
- delivery estimate,
- returns route,
- price/tax/shipping clarity,
- localized copy PL/EN/DE,
- product images with alt text.

## Rule
A premium UI cannot pretend the shop is ready when fulfilment is not wired. Preview mode is better than fake availability.

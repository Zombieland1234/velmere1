# Velmère PASS 122 — Product Truth / Size / Care Protocol

## Purpose
The product layer must stop being only a visual preview. Each garment needs structured truth data so the UI can show material, composition, fit, care, size guide, delivery note and return note before checkout.

## New product truth profile
Each product can now expose:
- material,
- composition,
- weight,
- fit,
- care instructions,
- size guide note,
- measurements,
- delivery note,
- return note,
- launch-control note.

## Why this matters
A clothing brand cannot rely on VLM, Shield or community systems to compensate for missing commerce basics. The product must be clear before any wallet, membership or checkout action.

## Current state
The first preview drop now has structured truth profiles, but the values are still marked as planned/preview where needed. They must be confirmed before public sale.

## Before public sale
Every product must have:
- confirmed composition,
- confirmed weight,
- production sample QA,
- shrinkage/care test,
- final measurement table,
- provider SKU and variant mapping,
- final delivery estimate,
- final return route.

## Rule
No product should become active while product truth is missing or fulfilment is disabled.
